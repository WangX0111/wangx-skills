# Mode C: 内容重构与深度改写详细规范

> 对传入的链接或内容进行深度分析，提取核心观点、关键论据和内容脉络，输出一篇「保留核心观点但表达方式完全重建」的全新原创文章。

---

## 六条改写原则

### 原则1：彻底重构表达

不做近义词替换式改写，完全摒弃原文的句式结构、段落顺序、比喻手法和修辞节奏，确保文章具有全新的语言风格和表达方式。

**反面示例**（近义词替换，禁止）：
```
原文：这种方法极大地提升了推理效率
改写：这一手段显著地改善了推理效能  ← 禁止
```

**正面示例**（彻底重构）：
```
原文：这种方法极大地提升了推理效率
改写：以前跑一次推理，KV cache 像个无底洞，现在三路记忆一层层压下来，FLOPs 直接砍到 27%  ← 允许
```

### 原则2：保留核心价值

严格保留核心科研价值、关键技术判断、重要结论和完整的论证链条，不得丢失关键信息。

**必须保留**：核心技术指标、关键实验结论、重要因果关系、方法论核心思路
**允许调整**：叙述顺序、类比方式、解释角度、信息密度分布

### 原则3：重新设计结构

完全重新设计文章整体结构，灵活调整：
- **开头切入点**：从背景、问题、案例、数据等不同角度切入
- **论证逻辑顺序**：因果关系、时间线、重要性排序等
- **标题层级体系**：采用新的组织方式
- **结尾总结方式**：归纳、展望、应用等

### 原则4：创新叙事主线

按照新的逻辑主线重新组织内容，常用叙事模式：

| 叙事模式 | 结构 | 适用场景 |
|:---------|:-----|:---------|
| 问题驱动型 | 问题定义 → 方案推导 → 设计原则 → 模块解释 → 意义总结 | 技术方案解读 |
| 挑战突破型 | 现状分析 → 挑战识别 → 解决思路 → 实现路径 → 效果评估 | 性能优化、架构演进 |
| 脉络梳理型 | 背景铺垫 → 核心创新 → 技术细节 → 应用场景 → 未来展望 | 产品发布、论文解读 |
| 逆向解构型 | 效果/结论先行 → 逐层回溯原因 → 揭示设计智慧 | 研究成果分析 |

### 原则5：增强独立性

允许并鼓励补充必要的上下文解释和背景信息，使文章成为一篇独立完整的原创技术分析，而非对原文的简单复述或改写。

补充方向：
- 前置知识补充（让读者无需读原文也能理解）
- 行业背景和对比（增加分析深度）
- 术语解释（降低阅读门槛）
- 实际影响分析（连接到读者关心的场景）

### 原则6：消除标志性特征

删除或彻底改写原文中高度标志性的比喻、特定口头表达、情绪化句式和个人化表达方式，确保改写后的文章在表达风格上与原文有明显区分。

---

## 重构红线

- 禁止近义词替换式改写
- 禁止保留原文标志性表达
- 必须保留核心论点和关键论据
- 必须保留完整论证链条

---

## 作者判断章节

在文章结尾加入独立的「作者自己的判断」章节，包含三个维度：

| 分析维度 | 内容 |
|:---------|:-----|
| **真正贡献** | 这项工作的真正核心贡献是什么，解决了什么关键问题 |
| **证据不足** | 目前证据还不充分或存在疑虑的地方，有哪些局限性 |
| **后续验证** | 后续最值得进行的消融实验或验证工作，下一步研究方向 |

> 此章节体现独立分析能力，非对原文观点的复述，而是基于重构理解后的原创判断。

## 对比展示格式

向用户展示原文与改写后文章的关键差异：

```
【原文核心段落】
...

【重构后版本】
...

【重构说明】
- 结构变化：从 XXX 结构调整为 XXX 结构
- 叙事变化：切入点从 XXX 转为 XXX
- 表达重建：句式、比喻、修辞完全重建
- 信息保留：核心论点 X/Y/Z 完整保留
- 信息增强：补充了 A/B/C 背景解释
```

## 多链接整合规则

当用户输入多个链接时，额外执行：

| 操作 | 说明 |
|:-----|:-----|
| **交叉验证** | 不同来源对同一论点的表述差异 |
| **信息整合** | 互补信息合并，重叠信息去重 |
| **矛盾标注** | 不同来源的矛盾之处，标注供用户确认 |
