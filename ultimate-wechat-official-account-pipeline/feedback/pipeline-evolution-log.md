## 进化机制说明

### 1. 反馈收集触发点

以下场景会自动收集用户反馈：

| 触发点 | 收集内容 | 存储位置 |
|:-------|:---------|:---------|
| 内容创作完成后 | 用户对文章质量的评价 | 本文件「内容质量反馈」 |
| 配图/封面生成后 | 用户对视觉效果的评价 | backend-ratings.yaml |
| 排版渲染完成后 | 用户对排版效果的评价 | backend-ratings.yaml |
| 发布完成后 | 用户对发布流程的评价 | backend-ratings.yaml |
| 全流程结束后 | 用户对整体体验的评价 | 本文件「整体体验反馈」 |

### 2. 进化维度

#### 2.1 路由策略进化

根据 backend-ratings.yaml 的数据，自动调整路由权重：

```
新权重 = 基础权重 × (成功率因子) × (质量评分因子) × (时效衰减因子)

成功率因子 = success_count / total_calls
质量评分因子 = avg_quality / 5
时效衰减因子 = exp(-天数/30)  # 近期反馈权重更高
```

**进化示例**：
- 初始：`baoyu-imagine` 权重 1.0
- 经过 10 次调用，8 次成功，平均质量 4.5 → 权重升至 1.2
- `xxb-qwen-image` 5 次调用，3 次成功，平均质量 3.0 → 权重降至 0.8
- 结果：下次用户说「配图要高质量的」→ 优先路由到 `baoyu-imagine`

#### 2.2 风格引擎进化

根据用户在不同风格预设下的满意度，调整风格推荐策略：

| 风格预设 | 使用次数 | 平均满意度 | 进化动作 |
|:---------|:---------|:-----------|:---------|
| khazix | 0 | 0 | 保持默认推荐 |
| formal | 0 | 0 | 保持默认推荐 |
| lifestyle | 0 | 0 | 保持默认推荐 |
| business | 0 | 0 | 保持默认推荐 |

**进化方向**：
- 某风格满意度持续高于其他 → 该账号默认风格自动迁移
- 用户多次手动切换风格 → 记录偏好，下次优先推荐

#### 2.3 质检标准进化

根据四层质检的实际拦截率和用户反馈，动态调整门槛：

```
如果 L1 拦截率 > 30% 且用户反馈「过于严格」→ 适度放宽阈值
如果 L3 拦截率 < 5% 且用户反馈「质量不稳定」→ 适度收紧阈值
```

#### 2.4 输出目录策略进化

跟踪用户最常使用的输出位置，优化自动推断逻辑：

| 用户习惯 | 检测方式 | 进化动作 |
|:---------|:---------|:---------|
| 多次指定同一目录 | 指令关键词匹配 | 建议设为账号默认输出目录 |
| 总是用自动推断 | 无显式指定 | 保持自动推断，优化 slug 生成规则 |
