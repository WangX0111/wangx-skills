# 微信公众号全链路内容运营引擎

> **极简心智模型**：输入 → 选择 → 输出。复杂判断交给 Agent，用户只需确认。

---

## 核心工作流

用户与 Pipeline 的交互只有 **6 个步骤**，每个步骤 Agent 都会给出默认推荐，用户可确认或调整：

```
用户输入（想法 / 内容 / 链接 / 文件）
    ↓
Step 1: Agent 判断内容来源类型，推荐处理策略
    ↓
Step 2: 风格选择（Agent 推荐默认风格，用户可切换）
    ↓
Step 3: 内容创作（Agent 执行，关键节点需用户确认）
    ↓
Step 4: 配图选择（Agent 推荐是否配图及效果，用户确认）
    ↓
Step 5: 排版选择（Agent 推荐主题，用户确认）
    ↓
Step 6: 输出到目录（自动生成完整文件包）
    ↓
（可选）Step 7: 一键发布
```

**用户负担最小化原则**：
- Agent 负责：内容来源判断、风格匹配、配图策略、主题推荐、后端路由
- 用户负责：确认 or 调整（每个步骤都可跳过默认，自定义选择）

## Step 1: 输入接收与来源判断

### 用户输入类型

| 输入形式 | 示例 | Agent 自动判断 |
|:---------|:-----|:---------------|
| **想法/关键词** | 「帮我写一篇 AI 对程序员影响的文章」| 从零创作 → 进入选题+创作流程 |
| **已有内容** | 用户粘贴草稿、文章、笔记 | 内容优化 → 诊断+改写/润色 |
| **链接** | 单个 URL 或多个 URL | 内容重构 → 深度分析+重建表达 |
| **文件** | PDF、Word、Markdown 文件 | 提取内容 → 按内容类型处理 |

**用户不需要理解「Mode A/B/C」**，Agent 根据输入自动选择内部处理策略，只需告知用户推荐方案（如「检测到你提供了链接，建议采用深度重构方式，保留核心观点但重新组织表达，是否继续？」）。

### 输出目录确定

执行第一步时即确定输出位置，按优先级：

| 优先级 | 来源 |
|:-------|:-----|
| 1 | 用户指令指定（「保存到 ~/Desktop/xxx」）|
| 2 | 输入文件 frontmatter 中的 `output_dir` |
| 3 | 账号配置 `output_dir` |
| 4 | 全局默认 `output_dir` |
| 5 | 自动推断：`./wechat-pipeline/{article-slug}/` |

确定后告知用户：**「所有产出将保存到 {output_dir}/」**。

---

## Step 2: 风格选择

Agent 根据内容主题和账号配置，推荐默认风格，展示选项供用户确认或切换。

### 可选风格

| 风格 | 适合内容 | 核心感受 |
|:-----|:---------|:---------|
| **卡兹克** | 科技/AI/产品评测 | 活人感、口语化、文化升维 |
| **正式** | 行业报告/深度科普 | 严谨、数据驱动、逻辑清晰 |
| **轻松** | 生活/情感/随笔 | 温暖、细腻、共鸣感强 |
| **商业** | 财经/管理/商业分析 | 洞察、案例、可执行建议 |
| **自定义** | 学习历史文章生成专属风格 | 你的独特调性 |

**交互示例**：
```
Agent：根据「AI 编程助手」主题，推荐「卡兹克」风格（科技类文章，口语化+文化升维）。
      可选：正式 / 轻松 / 商业 / 自定义
用户：就用卡兹克 → 确认
用户：换成正式的 → 切换
```

### 降味强度（可选调整）

Agent 默认根据账号配置选择降味等级，用户可调整：
- 1级：微调措辞，保留原结构
- 3级：重写段落，消除明显 AI 痕迹
- 5级：完全重写，像朋友聊天

---

## Step 3: 内容创作

Agent 根据 Step 1 判断的来源类型和 Step 2 选定的风格，执行内容创作。此阶段**关键节点需用户确认**。

### 处理策略（内部自动选择）

| 来源类型 | Agent 内部处理 | 用户确认节点 |
|:---------|:---------------|:-------------|
| **从零创作** | 选题策划 → 生成大纲 → 分段扩写 → 四层质检 | 大纲确认、初稿确认 |
| **内容优化** | 内容诊断 → 推荐优化方案 → 改写/润色 → 四层质检 | 诊断报告、优化方案、对比确认 |
| **内容重构** | 深度分析 → 全新结构设计 → 全文重构 → 四层质检 | 新结构确认、对比确认 |

### 四层质检（内部机制，不暴露给用户）

创作完成后自动运行质检，未通过则内部修复，不打扰用户：

- **L1 硬性规则**：禁用词/标点/套话扫描
- **L2 风格一致性**：开头/节奏/口语化确认
- **L3 内容质量**：观点支撑/知识输出/文化升维
- **L4 活人感终审**：温度感/独特性/心流

**评分门槛**：低于 70 分自动返回优化，不向用户暴露质检细节，只输出高质量结果。

---

## Step 4: 配图选择

Agent 分析文章内容，推荐配图方案，用户确认或调整。

### 交互流程

```
Agent：分析完成，建议在以下位置配图：
      1. 开篇：科技概念插画（AI生成）
      2. 中段：产品界面截图（搜索真实图片）
      3. 结尾：情绪氛围图（AI生成）
      是否生成配图？（是/否/调整）

用户：是 → 自动执行
用户：否 → 跳过配图，保留占位符
用户：调整 → 选择：只生成封面 / 只生成正文图 / 全部生成
```

### 配图效果选择

如需生成图片，Agent 推荐默认效果，用户可切换：

| 效果选项 | 说明 | 后端路由 |
|:---------|:-----|:---------|
| **高质量**（默认）| 细节丰富、风格多样 | baoyu-imagine |
| **中文友好** | 中文提示词理解好 | xxb-qwen-image |
| **图生图** | 基于参考图生成 | xxb-nanobanana2-image |
| **快速** | 低成本快速出图 | baoyu-imagine 轻量模式 |

### 封面选择

Agent 根据文章类型推荐封面风格，用户可切换：

| 封面选项 | 说明 | 后端路由 |
|:---------|:-----|:---------|
| **专业封面**（默认）| 11 调色板 × 7 渲染风格 × 3 比例 | baoyu-cover-image |
| **AI 自由生成** | 自定义提示词直接出图 | baoyu-imagine |

---

## Step 5: 排版选择

Agent 根据文章类型推荐排版主题，用户确认或切换。

### 主题推荐

| 文章类型 | Agent 推荐主题 | 风格 |
|:---------|:---------------|:-----|
| 科技类 | 科技蓝、赛博朋克、极客黑 | 简约、冷色调 |
| 财经类 | 经典蓝、玫瑰金、石墨黑 | 专业、稳重 |
| 情感类 | 人言兑、樱花粉、薰衣紫 | 温暖、治愈 |
| 生活类 | 春天、锤子便签、吉卜力 | 清新、文艺 |
| 教育类 | 清雅蓝、嫩青、森林 | 清新、知识感 |

**交互示例**：
```
Agent：推荐「科技蓝」主题，与你的 AI 工具评测内容匹配。
      可选：赛博朋克 / 极客黑 / 其他
用户：赛博朋克 → 切换
```

### 排版后端

- 默认：`baoyu-markdown-to-html`（94 种主题 + 微信兼容）
- 用户指定公众号助手风格：`official-account-assistant`

---

## Step 6: 输出

所有产出文件保存到 Step 1 确定的输出目录：

```
{output_dir}/
├── article.md          # 最终文章（Markdown）
├── article.html        # 排版后微信兼容 HTML
├── cover.png           # 封面图
├── images/             # 正文配图
└── meta.yaml           # 发布元数据（标题、摘要、作者）
```

告知用户：**「文件已保存到 {output_dir}/，包含文章、排版 HTML、封面和配图。」**

---

## Step 7: 发布（可选）

用户可选择直接发布或稍后手动发布。

### 发布通道选择

Agent 根据账号配置推荐默认通道，用户可切换：

| 通道 | 适用场景 |
|:-----|:---------|
| **API 发布**（默认）| 已配置 API，快速稳定 |
| **浏览器发布** | 无 API 配置，首次使用 |
| **公众号助手** | 需要降味+发布一体化 |

发布前自动终审：质量评分、配图完整性、封面尺寸、敏感词扫描。

## 输入类型判断逻辑（内部）

Agent 根据用户输入自动选择内部处理策略，**不暴露 Mode 名称给用户**：

| 用户表达 | 内部策略 | 用户感知 |
|:---------|:---------|:---------|
| 只有想法/关键词/选题 | 从零创作 | 「将为你从零创作」 |
| 「优化一下」「润色」「降味」+ 已有内容 | 内容优化（保留结构，局部调整）| 「将基于你的内容进行润色优化」 |
| 「深度改写」「重构」「用自己的话写」+ 链接/内容 | 内容重构（重建表达，全新结构）| 「将深度重构，保留核心观点但重新组织表达」 |
| 多个链接 + 「整合」 | 多源重构（交叉验证，整合输出）| 「将整合多篇文章，提取核心观点重新创作」 |

详细重构规范见 [references/mode-c-reconstruction.md](references/mode-c-reconstruction.md)。

## 内容评分标准（内部）

| 维度 | 权重 | 说明 |
|:-----|:-----|:-----|
| Hook 强度 | 25% | 开篇能否留住读者 |
| 价值密度 | 25% | 教学、娱乐或启发 |
| 平台适配 | 20% | 格式、长度、风格合适 |
| CTA 清晰度 | 15% | 下一步行动明确 |
| 视觉吸引力 | 15% | 配图和排版是否吸引 |

**门槛**：低于 70 分内部返回优化，不进入下一阶段。

## 重要原则

1. **用户心智极简**：Agent 负责复杂判断，用户只做选择题
2. **默认即最优**：每个步骤 Agent 给出的默认推荐都基于内容分析和历史数据
3. **确认机制**：关键环节（大纲、风格、配图方案、排版主题、发布）必须用户确认
4. **质量门禁**：内部四层质检，不达标自动修复，不暴露给用户
5. **保留核心**：优化/重构时绝不丢失原内容的核心信息和价值
6. **风格一致**：同一账号保持风格一致性
7. **输出可定**：所有产出统一保存到指定输出目录
8. **效果驱动**：配图、排版、发布都按效果选择，而非技术名称

## 可插拔后端能力层

每个能力域支持多后端，Agent 根据用户选择的效果自动路由：

| 能力域 | 用户可选效果 | 后端 Skill |
|:-------|:-------------|:-----------|
| AI 图像生成 | 高质量 / 中文友好 / 图生图 / 快速 | baoyu-imagine / xxb-qwen-image / xxb-nanobanana2-image |
| 封面设计 | 专业封面 / AI 自定义 | baoyu-cover-image / baoyu-imagine |
| 智能配图分析 | 结构化分析 | baoyu-article-illustrator |
| 排版渲染 | 微信主题 / 公众号助手 | baoyu-markdown-to-html / official-account-assistant |
| 发布 | API / 浏览器 / 助手 | baoyu-post-to-wechat / official-account-assistant |
| 资料搜索 | 网络搜索 | xxb-tavily-search |

用户永远不需要知道后端 Skill 名称，只需选择效果（如「高质量的」）。

## 自进化机制

Pipeline 每次执行后根据用户反馈自动学习：

- **路由优化**：根据历史成功率调整后端权重，越用越精准
- **风格偏好**：跟踪用户风格选择，优先推荐高频偏好
- **质检调优**：根据拦截率和用户反馈动态调整阈值
- **日志位置**：`feedback/backend-ratings.yaml`、`feedback/pipeline-evolution-log.md`

详见 `feedback/pipeline-evolution-log.md`。
